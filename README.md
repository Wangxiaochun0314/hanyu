# 汉语

汉语计划：用母语写代码，用母语训练模型。

目标  
让中国开发者在最前沿的 AI 领域，能够全程使用中文变量名、函数名、注释和文档进行研发、协作与交付，形成可闭环、可传承、可审查的中文代码生态。

为什么现在做这件事
1. 2025 年，中国训练了全球最强的多个开源中文大模型，但 98% 以上的核心代码仍然使用英文标识符。  
2. 变量名和函数名是知识的最小单元，用英文写＝知识表达权被外语垄断。  
3. 安全、可控、可审计的国家战略级系统，需要彻底可读的母语代码。  
4. Unicode 早就是标准，Python/Rust/JavaScript 等主流语言早已原生支持中文标识符，技术上毫无障碍。  
   唯一缺的只是“第一个吃螃蟹的人”。

本仓库规则（极简三条）
1. 变量名、函数名、类名、包名必须使用有意义的中文（拼音、英文、中英混写均不允许）。  
2. 注释和文档必须全中文。  
3. 鼓励提交训练、推理、部署、数据清洗等完整可运行的 AI 项目。

我们不吵架，不骂人，只提交代码。  
用事实和可运行的结果说话。

欢迎每一位愿意用母语思考的开发者加入。  
一起把中国 AI 的底层代码主权拿回来。

—— 2025 年 12 月启动
